Oh dear.

There are a couple of reasons why you might be seeing this:

  1. The site owner's web root is configured incorrectly

    Firefolio is configured to be secure by default. As such, the bits and pieces that make it work are seperated out from the things you need to see the end result, so:

    • Please tell them (or yourself) that any site using Firefolio should point to the public_html folder, and not the firefolio-x.x.x folder included in the download.
    • If you can, ask them to read the documentation in case they get stuck.
  2. A 'bad actor' has compromised the site/server

    This is unlikely, but totally possible that a weak point has allowed someone to do bad things behind the scenes. In which case:

    • Contact the host.
    • Contact the developer.
      • You should only do this if this message didn't show up before.
      • If you think you know what's wrong, tell them.
      • As per the MIT Open Source License, the developer can't be held liable for any damage, but you have their heartfelt apologies!

In any case, don't panic! The actual site should still be available here.
